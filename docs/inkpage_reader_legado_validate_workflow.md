# Inkpage Reader x Legado Validate Workflow

## 角色

這是由 main workflow 內部路由的 agent workflow。使用者不需要知道這個 workflow 存在。

用於檢查、review、重現、驗證與風險評估，特別是使用者不是要求立即實作時。

## 內部推理層

不要把這層輸出給使用者。

1. 保留驗證問題、預期行為或風險。
1. 接收任務與 main workflow 已讀過的 index 摘要。
1. 選擇最相關的模組文件。
1. 內部確認驗證範圍：
   - 要驗證的具體行為或假設。
   - 相關邊界與下游影響。
1. 收集證據：
   - 讀相關程式碼、測試、設定或文件。
   - 只讀回答驗證問題所需的最小內容。
1. 內部分開記錄：
   - 有證據支持的結論。
   - 仍無法確認的部分。
1. 如果和 Legado 對照，確認對照範圍不超出本專案現有小說閱讀功能。

## 對外回報層

1. 用下面的 Before / After 格式回報。
1. 如果驗證顯示需要修正，詢問使用者是否要繼續進入修改流程；不要直接開始改檔案。

## 回報規則

- 使用者溝通使用繁體中文與白話說明。
- 使用者回報中不要暴露模組名、檔案路徑、函式名或程式碼片段。
- Before / After 是唯一的人類確認介面。
- 技術細節留在內部推理。

## Before / After 格式

**Before**：用一到三句白話說明正在驗證什麼，以及驗證前哪裡不確定或有風險。

**After**：用一到三句白話說明驗證結果，或仍無法確認什麼。

任何後續改檔案操作前，都要等待使用者明確確認。

## Atlas 更新條件

只有當驗證發現既有 atlas 事實不正確，例如模組邊界、所有權或外部 API 描述錯誤時，才更新 atlas。新發現的風險先回報使用者，不要因一般驗證就寫回 atlas。

## 交付方式

一般工作不自動 commit 或 push；只有使用者明確要求提交、推送，或正在走發布流程時才做。
