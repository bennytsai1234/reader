# Reader x Legado 介紹工作流程

## 目的

- 用於使用者想快速理解 `reader` 是什麼、做什麼、主要邊界在哪裡。
- 輸出保持白話、使用者可讀；除非使用者要求，不要變成架構報告、檔案清單或模組 checklist。
- 介紹流程不修改程式碼。
- 一般介紹不要重新執行 Codebase Atlas；使用本工作流程與既有 Atlas 文檔即可。

## 工作流程

1. 打開 [reader_legado_index.md](reader_legado_index.md)，閱讀目的、模組摘要、參考邊界與工作流程列表。
2. 只閱讀能回答問題的模組文檔，不為了完整性展開全部模組。
3. 若 Atlas 不足以解釋使用者可見行為，再檢查必要程式碼或專案文檔。
4. 用白話說明：
   - 這個專案是什麼。
   - 服務的使用者或任務是什麼。
   - 主要工作包含哪些。
   - 重要責任邊界，只有在有助理解時才說。
5. 如果使用者要求深入技術脈絡，建議切到調查工作流程。
6. 若介紹過程發現 Atlas 過時，只記錄需要更新的文檔；不要在未確認時重建 Atlas。
7. 若沒有檔案變更，摘要說明不需要 commit；若有文件變更，依交付政策完成。

## 介紹摘要

最後輸出應先給最簡單有用的解釋，再用一小段說主要 moving parts。避免列出每個檔案、每個依賴或每個模組。

## 交付政策

提交並推送。若有檔案變更，完成驗證後建立聚焦 commit，確認目標 remote/branch 後推送；若 remote 或 branch 不清楚，推送前必須先詢問。若沒有檔案變更，摘要結果並說明不需要 commit。
