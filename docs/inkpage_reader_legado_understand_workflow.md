# Inkpage Reader x Legado Understand Workflow

## 角色

這是由 main workflow 內部路由的 agent workflow。使用者不需要知道這個 workflow 存在。

用於專案介紹、說明、所有權問題、可行性問題與調查。

## 內部推理層

不要把這層輸出給使用者。

1. 保留使用者的問題或症狀。
1. 接收 main workflow 已讀過的 index 摘要。
1. 選擇最相關的模組文件；不要預設讀完所有模組文件。
1. 只有在模組脈絡不足以準確回答時，才讀程式碼、測試、文件、log 或設定。
1. 內部分開記錄：
   - 由文件或程式碼支持的已確認事實。
   - 尚未驗證但合理的假設。
   - 仍未知的部分。
1. 如果問題涉及 Legado，只使用 index 中允許的部分參考邊界。

## 對外回報層

1. 用白話回答使用者問題。
1. 如果有假設或未知，直接說明不確定之處。
1. 如果需要後續實作，詢問使用者是否要繼續進入修改流程。

## 回報規則

- 使用者溝通使用繁體中文與白話說明。
- 使用者回報中不要暴露模組名、檔案路徑、函式名或程式碼片段。
- 技術細節留在內部推理。
- Before / After 是唯一的人類確認介面。

## Before / After 格式

**Before**：用一到三句白話說明目前狀態，以及哪裡錯誤、不清楚、缺失或有風險。

**After**：用一到三句白話說明操作完成後會變成什麼狀態。

任何後續改檔案操作前，都要等待使用者明確確認。

## Atlas 更新條件

只有當調查發現既有 atlas 事實不正確，例如模組邊界、所有權或外部 API 描述錯誤時，才更新 atlas。新發現的風險先回報使用者，不要因一般調查就寫回 atlas。

## 交付方式

一般工作不自動 commit 或 push；只有使用者明確要求提交、推送，或正在走發布流程時才做。
