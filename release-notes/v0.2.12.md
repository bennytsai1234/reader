# 0.2.12 (2026-04-20)

這個版本集中收尾閱讀器最後幾個明顯的交互問題，並把自動翻頁重新拆成真正可用的兩種模式。另一條重點是壓住 `酷我小說` 這類重型 JS 書源在校驗 discovery 規則時的崩潰風險。

## 亮點

- 閱讀正文不再在頂部額外留一塊空白，版面回到更貼近預期的閱讀狀態
- 自動翻頁正式拆成 `平移翻頁` 和 `上下滾動` 兩條執行路徑，滾動模式改成真正連續位移
- 書源校驗對重型 JS `exploreUrl` 新增序列化與 timeout 清理，降低 `酷我小說` 這類來源帶來的 crash 風險

## 詳細更新

### 閱讀器

- 修正正文上方仍保留額外閱讀留白的問題，現在正文直接貼齊頂部
- 修正「上一章」按鈕會落到上一章末尾的行為，改為回到上一章第一列
- 移除未完成的 `自定義背景圖` 入口與對應閱讀頁渲染
- 白天 / 夜間主題切換改成只切主題，不再觸發整套 repaginate，減少微跳頁
- 上下滾動模式下的頁碼與百分比改為依據實際閱讀焦點位置計算

### 自動翻頁

- `平移翻頁` 保留逐頁掃描線式節奏
- `上下滾動` 改成直接推動 scrollable 連續滾動，不再用章節跳轉模擬
- 切換模式、調整速度時會重啟自動翻頁週期
- 到達書尾時會正確停止，不再維持假啟動狀態

### 書源管理 / 校驗

- 修正書源管理頁全選後，底部批量列的三點選單被文案擠壓而難以點擊的問題
- `ExploreUrlParser` 對 JS 型 discovery 規則改成序列化執行，避免多條重型來源同時壓住 JS runtime
- async JS 失敗後不再補跑 sync fallback
- JS discovery 規則現在支援 timeout 後先 dispose runtime 再回報錯誤
