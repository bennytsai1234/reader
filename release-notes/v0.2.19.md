# 0.2.19 (2026-04-23)

這個版本集中收斂 Reader 最近一輪「書源管理 / 發現頁 / 校驗穩定性」的調整。重點不是新增大型功能，而是讓混合書源包在匯入與管理上更誠實、發現頁對 Legacy Legado 規則更相容，同時把批量校驗時最可疑的資源釋放與內層併發風險先壓下來。

## 亮點

- 書源管理不再把不支援來源默默吞掉，混合來源包的數量與預覽更接近實際內容
- 發現頁改善舊式 Legado 規則相容性，修回一批 JS / exploreUrl 來源
- 校驗流程補上 `AnalyzeRule` 釋放，並把校驗模式的內層多頁抓取併發降到 1，降低高壓校驗時的資源風險

## 詳細更新

### 書源管理與發現頁

- `SourceType` 對齊 Legado 類型定義，整理 `book / audio / image / file` 語意
- 匯入預覽與書源管理會保留不支援來源，但自動停用並加上標記，而不是直接消失
- `check all` 改成真正針對目前已存入的全部書源，不再只受目前畫面過濾條件影響
- 發現頁來源篩選改用更一致的 runtime gating，避免不支援來源混入 discovery 清單
- 發現頁分類區塊從固定格線改成較穩定的 wrap 版面
- Explore 結果頁補上 stale request 防護與取消行為，避免 refresh/dispose 後舊結果回寫

### Explore / 規則相容性

- `ExploreUrlParser` 補強對舊式 JS 規則的相容性
- 支援 legacy sync IIFE + async helper 的 explore JS 寫法
- 正規化 bare destructuring arrow params（例如 `[a, b]=>{}`）
- explore JSON 允許 trailing commas，修回部分實際來源包中的寬鬆 payload

### 校驗穩定性

- `BookInfoParser`、`BookListParser`、`ChapterListParser`、`ContentParser`、`WebBook` 相關 validation 熱路徑補上 `AnalyzeRule.dispose()`
- `loginCheckJs`、`preUpdateToc`、TOC `formatJs` 與 batch validation helper 的 `AnalyzeRule` 生命週期也補齊
- 校驗模式下的 TOC / content 多頁抓取改用 `pageConcurrency = 1`，避免 source 內部再展開多層並發
- 預設校驗範圍與最外層 source-level 並發 (`maxConcurrent = 5`) 維持不變，方便觀察這一輪穩定性修補的實際效果
