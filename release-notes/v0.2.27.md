# 0.2.27 (2026-04-26)

這個版本修正 scroll mode restore 完成後仍可能鎖住滾動的問題，並保留實際量測到的章內像素位置，降低重開後位置偏移。同時把部分啟動初始化延後到第一幀或首頁轉場後，減少 splash 銜接到主畫面時的卡頓。

## 修正

- scroll restore runner 完成後會主動 settle restore navigation，避免重開後 scroll input 被永久鎖住
- scroll mode 持久化會保留實際量測的 `localOffsetSnapshot`，不再用 `charOffset` 反推覆蓋精確像素座標
- 同一個 `charOffset` 下若 `localOffsetSnapshot` 改變，退出與 flush 仍會重寫 reader anchor
- progress debounce / flush 會攜帶 scroll local offset snapshot，提升重開位置一致性

## 啟動

- splash 只等待必要主題初始化，預設資料匯入、maintenance、簡繁字典預熱延後到首頁轉場後
- `Workmanager.initialize()` 與 debug log prefs 延後到第一幀後執行
- 自訂 welcome 背景圖檔案檢查改為 async，避免 build 中同步 I/O

## 驗證

- `flutter analyze`
- `tool/flutter_test_with_quickjs.sh --reporter expanded`
- `flutter test test/features/reader --reporter expanded`
