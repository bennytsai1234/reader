# 0.2.21 (2026-04-24)

這個版本是一次以閱讀位置恢復穩定性為主的 patch release。重點不是再擴大功能面，而是把最近補進 reader 的 precise resume 能力真正收斂到可發版的狀態，避免 scroll / slide 在不同啟動時機、不同快取視窗與換源流程下出現錯頁或錯列還原。

## 亮點

- scroll restore 不再因為 restore pending 而整頁空白；有內容可畫時會先顯示，再做二段精修
- slide reopen 改成以 durable anchor 為主，不再誤用舊的 window-global page index
- 本機精準錨點正式落盤，scroll mode 在 layout 未變時可更準確回到原本閱讀列
- precise anchor 的驗證與遷移流程補齊，加入書架、換源與 reopen 都不再輕易丟失精準恢復資訊

## 詳細更新

### 閱讀器 restore 與 precise resume

- scroll restore 改成 soft restore policy：有可見內容就先渲染，restore 中只保留輕量 overlay，不再用 hard block 把整頁鎖成背景色
- slide startup restore 與 mid-scroll recenter 分流，startup 只信 durable anchor / precise snapshot，不再優先沿用舊的 mapped window index
- `SlidePageController` 修正為在 `PageView` 尚未 attach 時保留 pending jump，避免初始化那一幀直接丟失目標頁
- `ReaderAnchor` snapshot 現在會持久化到本地進度欄位，並在 reopen 時依 layout 與內容指紋決定是否可重用

### 精準錨點有效性與進度保存

- `ReaderOpenTarget.resume()` 不再把舊 anchor 的 snapshot metadata 強行套到新的 durable location，避免同章節但不同 offset 時回到過期位置
- scroll precise restore 的內容驗證改成使用頁面/行結構指紋，而不是只比對粗粒度 pageCount/contentLength，降低 same-length source update 造成的錯誤重用
- progress-only update、加入書架流程與 source-switch 遷移現在都會保留 `readerAnchorJson`，不再把剛寫入的精準錨點覆蓋掉
- source switch 的 `migrateTo()` / `overwriteFrom()` 已攜帶 precise anchor 欄位，換源後若 durable location 未變，下一次 reopen 仍可復用本機精準恢復資訊

### 測試

- 新增 stale slide window、stale durable/anchor merge、chapter structure fingerprint 與 source-switch anchor preservation 的回歸測試
- reader restore、progress persistence、page attach deferred jump 與 page-level reopen flow 都補上對應驗證
